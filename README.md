# SPITE QTE 助手

> 基于 OpenCV 的图像识别脚本，自动完成 SPITE 游戏中的 QTE（快速反应事件）小游戏，解放双手。

![Python](https://img.shields.io/badge/Python-3.8+-blue.svg)
![OpenCV](https://img.shields.io/badge/OpenCV-4.x-green.svg)

---

## 📌 项目简介

本脚本通过 **屏幕实时捕捉 + HSV 颜色识别**，检测 QTE 判定条中的 **绿色安全区** 与 **白色判定线**，并在最佳时机自动按下 **空格键**，实现全自动 QTE 通关。[我实在是受不了这该死游戏的QTE了]

该脚本并非完美实现，单纯的受不了了，借助ai解放一下双手，如果有更高的需求可以自行借助ai补充。

---

## 🎮 适用场景

- SPITE 内的 QTE 节奏小游戏

## 🚀 使用说明

- 将游戏设置为 **全屏**，此脚本基于2k分辨率实现。
- 建议使用 PyCharm 导入该项目并使用 PyCharm 启动脚本
- 启动后根据代码提示"于 5s 内切换至游戏内 QTE 游戏界面"

- 控制台会实时显示判定信息：

  ```
  竖线: 1245 | 绿色: [980, 1520] | 距离: 28
  ```

- 触发成功提示：

  ```
  🎯 触发！竖线: 1245, 绿色中心: 1250
  ⌨️ 已按空格，冷却0.5秒
  ```

- 该脚本并未实现全完美判定。实在是太懒(bushi

## 🧠 工作原理

```text
屏幕捕获 (mss)
        ↓
HSV 色彩空间转换
        ↓
识别绿色判定区域（安全区）
识别白色判定线（指针）
        ↓
计算白线与绿区中心距离
        ↓
距离 < 提前量 → 自动按空格
```

## 🛠 配置参数说明

| 参数          | 说明                    | 默认值                            |
| :------------ | :---------------------- | :-------------------------------- |
| `BAR`         | QTE 判定条截图区域 (2K) | `{"top": 1060, "left": 120, ...}` |
| `GREEN_LO/HI` | 绿色判定区 HSV 范围     | `[25, 90, 95] ~ [45, 255, 255]`   |
| `WHITE_LO/HI` | 白色判定线 HSV 范围     | `[0, 0, 200] ~ [180, 30, 255]`    |
| `LEAD_PIXELS` | 提前触发像素            | `62` (约 2 帧)                    |
| `cooldown`    | 触发后冷却时间          | `0.5 秒`                          |

⚠️ **不同分辨率 / 画质需自行微调参数**

---

## 🖥 分辨率适配

如果你的分辨率不是 **2560×1440 (2K)**：

1. 使用截图工具测量 QTE 条的位置。
2. 修改 `BAR` 字典中的 `top`, `left`, `width`, `height`。
3. 根据实际测试自行调整 `LEAD_PIXELS`。
